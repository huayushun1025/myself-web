# 🚀 飞书音频会议纪要机器人：让 AI 为你开会做笔记

在日常工作中，大量的会议往往伴随着冗长的录音和耗时的纪要整理工作。为了解决这个痛点，我开发了这款**飞书音频会议纪要机器人**。它巧妙地将企业协作平台（飞书）、顶尖的语音识别技术（科大讯飞）以及强大的大语言模型（Kimi）结合在了一起，打造了一个全自动的“AI 会议秘书”。

## 💡 核心价值

只需将会议录音或语音消息转发给飞书机器人，它就能在后台自动提取文字，并经过 AI 的理解与润色，将逻辑清晰、结构化的会议纪要直接推送到你的飞书对话框中。**整个过程零人工干预，大幅节省精力，让团队成员能够更专注于会议本身的深度讨论。**

## ✨ 核心功能与亮点

- **全自动无缝接入**：深度集成飞书开放平台，通过 Webhook 实时监听和处理用户或群组中的音频/文件消息。完美支持飞书的 AES-CBC 事件加密订阅机制，保障数据安全。
- **高精度长语音转写**：对接科大讯飞 LFASR（录音文件识别）API，支持超长会议音频的高准度转写。
- **智能结构化总结**：引入 MiniMax (Kimi / Moonshot AI) 大语言模型，不再是简单的文字堆砌，而是按“核心议题”、“详细讨论”、“关键结论”和“待办事项(Todo)”自动提炼结构化 Markdown 输出。
- **异步响应，极佳体验**：基于 FastAPI 的异步处理架构（Background Tasks）构建微服务，接收任务后秒级响应“处理中”提示，完成后主动推送结果，避免超时断连。

## 🛠 技术架构与实现

本项目作为一个轻量级高并发的微服务，采用了以下技术栈：

- **后端框架**：`Python` + `FastAPI`，利用其原生的异步特性处理 Webhook 请求与网络 I/O。
- **API 组合**：
  - **飞书开放平台 API**：实现鉴权、机器人发信、多媒体文件下载。
  - **科大讯飞 API**：处理音视频文件的异步切片上传与轮询识别。
  - **Kimi (Moonshot AI) API**：Prompt Engineering 设计，控制输出纪要模板。
- **安全与环境**：支持 Pydantic Settings 进行环境变量的管理，并严谨实现了飞书 Webhook 回调的验签与解密。

## ⚙️ 它是如何工作的？

1. **触发机制**：用户在飞书中将一段冗长的会议录音发给机器人。
2. **下载与识别**：机器人服务接收事件后，安全地从飞书服务器下载音频文件，并通过 API 提交给科大讯飞进行语音转文字。
3. **AI 提炼**：获取到长篇转写文本后，将其作为 Context 喂给 Kimi 大模型，根据预设的系统提示词（System Prompt）进行逻辑梳理和摘要。
4. **结果回传**：最终生成的精美 Markdown 纪要将被转化为飞书消息格式，直接推送给对应用户。

## 🌟 总结回顾

这个项目不仅是一个实用的工具，也是一次将多个 AI 能力模块化组装的最佳实践。在开发过程中，重点攻克了**飞书加密事件解析**、**流式网络回调及长连接处理**等集成难题。目前这套系统已展现出快速部署和极强的可扩展性，未来还能进一步增加对多语种翻译、飞书云文档直接生成等进阶功能的支持。
